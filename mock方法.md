# 通用做题流程

## 第一步：先看类型签名
先把类型签名翻译成人话。

例如：

```haskell
prune :: Int -> Rose a -> Rose a
```

翻译：

- 输入一个 `Int`
- 输入一棵 `Rose a`
- 返回一棵 `Rose a`

所以这是一道：

## 构造题

因为输出不是 `Bool`，而是一个新的结构。

再例如：

```haskell
isNBranching :: Int -> Rose a -> Bool
```

翻译：

- 输入一个 `Int`
- 输入一棵树
- 返回 `Bool`

所以这是一道：

## 判断题

再例如：

```haskell
circuit :: Expr -> Circuit
```

翻译：

- 输入一种结构 `Expr`
- 输出另一种结构 `Circuit`

所以这是一道：

## 翻译题 / 结构转换题

---

## 第二步：看输入类型有几个构造器
输入类型有几个构造器，函数通常就至少分几种情况写。

例如：

```haskell
data Rose a = Leaf a | Branch [Rose a]
```

那就先写骨架：

```haskell
f (Leaf x) = ...
f (Branch xs) = ...
```

再例如：

```haskell
data Expr = Var Char
          | Not Expr
          | And Expr Expr
          | Or Expr Expr
          | Implies Expr Expr
```

那就先写：

```haskell
f (Var c) = ...
f (Not e) = ...
f (And e1 e2) = ...
f (Or e1 e2) = ...
f (Implies e1 e2) = ...
```

---

## 第三步：先写骨架，不要急着写细节
这一点非常重要。

例如做 `circuit` 题时，先写：

```haskell
circuit (Var c) = ...
circuit (Not e) = ...
circuit (And e1 e2) = ...
circuit (Or e1 e2) = ...
circuit (Implies e1 e2) = ...
```

即使右边全是 `...`，也比脑子一团乱要强得多。

因为这样我已经把一道题拆成几个小格子了。

---

## 第四步：每个分支先写中文，不要直接写 Haskell
例如 `prune` 这题：

### `Leaf`
中文思路：

- 叶子没有孩子
- 不需要剪
- 原样返回

### `Branch xs`
中文思路：

- 先保留前 `n` 个孩子
- 再对子树递归做 `prune`
- 最后重新包成 `Branch`

等中文思路清楚以后，再翻译成代码。

---

## 第五步：找递归点
这是最容易写错的地方。

一定要问自己：

## 递归到底是发生在“整体”上，还是发生在“更小的子部分”上？

通常来说：

- 列表递归：对 `xs` 递归
- 树递归：对子树递归
- 表达式递归：对 `e`、`e1`、`e2` 递归

不是对原来的整块结构原地再来一次。

如果写成这种形式，就要警惕：

```haskell
f (Branch xs) = f (Branch ...)
```

这很可能会原地打转，导致无限递归。

---

## 第六步：写完后做四个检查

### 1. 类型对不对？
例如：

- `Not` 里面放的是不是 `Expr`
- `Branch` 里面放的是不是 `[Rose a]`
- `prune` 的右边最后是不是 `Rose a`

### 2. 结构拆开后，有没有包回去？
例如拆了：

```haskell
Branch xs
```

最后是不是还写回：

```haskell
Branch (...)
```

### 3. 递归有没有真的变小？
例如：

- 是对 `xs` 递归
- 是对子树递归
- 是对子表达式递归

而不是对原结构原地重复。

### 4. 有没有自己定义自己？
例如：

```haskell
p4 = Not p4
```

这种写法就很危险，通常说明思路有问题。

---

# 我至少要建立的几个答题模板

## 模板 1：判断型树题
```haskell
f (Leaf x) = True/False/某条件
f (Branch xs) = 当前条件 && all f xs
```

---

## 模板 2：构造型树题
```haskell
f (Leaf x) = Leaf ...
f (Branch xs) = Branch (map f xs)
```

如果题目要求限制、裁剪、过滤，就在 `xs` 上先加：

- `take`
- `drop`
- `filter`

例如：

```haskell
f (Branch xs) = Branch (map f (take n xs))
```

---

## 模板 3：列表递归题
```haskell
f [] = ...
f (x:xs) = ...
```

---

## 模板 4：表达式翻译题
```haskell
f (Var x) = ...
f (Not e) = ...
f (And e1 e2) = ...
f (Or e1 e2) = ...
f (Implies e1 e2) = ...
```

本质是：

- 按构造器分情况
- 递归处理子结构
- 把一种结构翻译成另一种结构

# 以后每做一道题，我都先问自己这六个问题

## 1. 这题是什么题型？
是判断题、构造题，还是翻译题？

## 2. 输出类型告诉了我什么？
如果输出是 `Bool`，通常是判断题。  
如果输出和输入结构同类，通常是构造题。  
如果输出是另一种类型，通常是翻译题。

## 3. 输入类型有几个构造器？
有几个构造器，通常就有几个 pattern matching 分支。

## 4. 每个构造器的中文处理思路是什么？
先不要急着写代码，先写人话。

## 5. 递归点在哪里？
是对列表尾巴、子树、还是子表达式递归？

## 6. 写完后我有没有检查类型、结构、递归？
这是避免低级错误的关键。
